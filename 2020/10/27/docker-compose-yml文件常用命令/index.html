docker-compose.yml文件常用命令

发表于 2020-10-27

常用命令

environment

设置环境变量，后续可通过getenv()函数获取该变量。

image

在此镜像的基础上

指定为镜像名称或镜像 ID。如果镜像在本地不存在，Compose 将会尝试拉取这个镜像。

container_name

指定容器名称。默认将会使用 项目名称_服务名称_序号 这样的格式。

这里${USER}-${PORT}-${PORT}是.env文件里配置的，.env文件要存放在启动docker-compose.yml文件一致的位置。

entrypoint

指定服务容器启动后执行的入口文件。这里是openresty容器的启动脚本(nginx)

start-openresty.sh脚本中：

nginx默认启动是在daemon模式下，所以在使用命令docker run -d nginx /usr/sbin/nginx时，容器启动nginx后会立刻退出，所以需要使用nginx的前台运行模式，需要在配置文件中加“daemon off”指令，或在启动时加“daemon off;”参数。注意off后面的分号不要忽略。

https://hub.docker.com/_/nginx/

tty

模拟一个伪终端,用docker-compose up -d启动起来的容器不会立即停止

docker-compose up -d 发现容器启动后又停止

模拟一个伪终端，阻止容器退出

volumes

挂载一个目录或者一个已存在的数据卷容器。

数据卷的格式可以是下面多种形式：

链接：https://www.jianshu.com/p/4f14637f4b35

links

服务之间可以使用服务名称相互访问，links 允许定义一个别名，从而使用该别名访问其它服务

networks

配置容器连接的网络。

使用已存在的网络

ports

暴露端口信息。

使用宿主端口：容器端口 (HOST:CONTAINER) 格式，或者仅仅指定容器的端口（宿主将会随机选择端口）都可以。
